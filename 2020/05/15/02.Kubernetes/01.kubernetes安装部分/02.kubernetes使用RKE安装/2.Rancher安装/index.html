2.Rancher安装

HeiYueQiMa 2020-05-15 07:38:25
Kubernetes > Rancher

Racher安装

rancher一个很好用的图形界面管理工具

比 KubeSphere 好用一点 里面很多的helm charts可以直接用的

一. 安装cert-manager

这个是可以利用ingress自动签名证书的 当前最新版本是v0.15

不得不说这个工具是真好用 就是有时候不是那么灵

cert-manger文档地址

这里我采用的是直接安装的方式 没有使用helm
我这里安装的是v0.12的方式 这个版本好用一点 v0.15用的我难受
一条命令安装完事 前提需要安装好kubectl

kubectl apply --validate=false -f https://github.com/jetstack/cert-manager/releases/download/v0.12.0/cert-manager.yaml

安装完成后 执行命令 查看容器是否创建成功
可以创建一个证书测试一下

kubectl get pod -n cert-manager

生成证书 需要提前准备2个文件

  1. 创建clusterissuer文件

生成证书的前提是 dns要解析到集群的任何一个IP地址上面

我这里是给rancher生成证书 rancher的helm安装 是安装不好证书的
这个好像是因为他使用的issuer是测试的 不是生产环境的 现在已经不给签发证书了 说因为什么速率限制啥的

这里的email需要修改一下 修改成你自己的邮箱

apiVersion: cert-manager.io/v1alpha2
kind: ClusterIssuer
metadata:
  name: letsencrypt-prod
spec:
  acme:
    server: https://acme-v02.api.letsencrypt.org/directory
    email: youmail@gmail.com
    privateKeySecretRef:
      name: letsencrypt-prod
    solvers:
      - http01:
          ingress:
            class: nginx

提交issuer

kubectl apply -f clusterissuer.yaml
  1. 创建证书文件

这个步骤不是必须的 用helm的时候 也可以自己签发证书的

apiVersion: cert-manager.io/v1alpha2
kind: Certificate
metadata:
  name: rancher.xxx.com
  namespace: cattle-system
spec:
  secretName: cert-rancher
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  commonName: rancher.xxx.com
  dnsNames:
  - rancher.xxxx.com
  acme:
    config:
      - http01:
          ingressClass: nginx
        domains:
          - rancher.xxx.com

提交证书

kubectl create ns cattle-system
kubectl apply -f cert.yaml  --valiate=false

查询证书状态 可以用

kubectl get cert -A

二. Helm安装

这里安装的是helm3 helm3比较简单一些 开始准备安装

helm3官方文档

helm3 release页面

首先去release页面下载一下二进制包

wget https://get.helm.sh/helm-v3.2.1-linux-amd64.tar.gz
tar xf helm-v3.2.1-linux-amd64.tar.gz
mv linux-amd64/helm /usr/local/bin

helm就安装完了 helm有很多的用途 后续专门写文档

三. 安装rancher

现在添加rancher 的 helm 仓库

helm repo add rancher-stable https://releases.rancher.com/server-charts/stable

安装rancher命令 这个还是可以优化一下的 具体 可以拉下helm的chart进行查看

这里需要修改 hostname email 的值

helm install rancher rancher-stable/rancher \
 --namespace cattle-system \
 --set hostname=rancher.xxxx.com \
 --set ingress.tls.source=letsEncrypt \
 --set letsEncrypt.email=youmail@gmail.com \
 --set letsEncrypt.environment=production \
 --set letsEncrypt.ingress.class=nginx

如果cert-manager没有问题的话 证书和ingress都会自己创建完成

四. Rancher的ingress问题处理

用helm安装好rancher以后 还不能直接打开 因为rancher的容器内开启了 80跳转443

这是我容器测试的结果

root@rancher-66b5cfc7f5-79wql:/var/lib/rancher# curl http://127.0.0.1
<a href="https://127.0.0.1/">Found</a>.

root@rancher-66b5cfc7f5-79wql:/var/lib/rancher#

我们这里要加一个 注解 还有修改服务的端口

kubectl edit service rancher -n cattle-system

这是修改前的状态

spec:
  clusterIP: 10.43.184.122
  ports:
  - name: http
    port: 80
    protocol: TCP
    targetPort: 80

修改后状态

spec:
  clusterIP: 10.43.184.122
  ports:
  - name: https
    port: 443
    protocol: TCP
    targetPort: 443

然后修改ingress的端口

kubectl edit ingress rancher -n cattle-system

修改前

spec:
  rules:
  - host: rancher.xxx.com
    http:
      paths:
      - backend:
          serviceName: rancher
          servicePort: 80

修改后

spec:
  rules:
  - host: rancher.xxx.com
    http:
      paths:
      - backend:
          serviceName: rancher
          servicePort: 443

给ingess添加下面这条注解

这条注解的目的是访问后端的协议是https

nginx.ingress.kubernetes.io/backend-protocol: "HTTPS"

加上以后就可以访问rancher了
