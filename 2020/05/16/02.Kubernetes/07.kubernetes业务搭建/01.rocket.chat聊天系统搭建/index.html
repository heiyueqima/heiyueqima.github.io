二. rocket.chat搭建过程

创建namespace

kubectl create ns rocket

准备证书文件 ingress上面会用

需要提前装好cert-manager 并且创建集群issuer [letsencrypt-prod]

apiVersion: cert-manager.io/v1alpha2
kind: Certificate
metadata:
  name: chat.xxxx.com
  namespace: rocket
spec:
  secretName: cert-chat
  issuerRef:
    name: letsencrypt-prod
    kind: ClusterIssuer
  commonName: chat.xxxx.com
  dnsNames:
  - chat.xxxx.com
  acme:
    config:
      - http01:
          ingressClass: nginx
        domains:
          - chat.xxxx.com

创建俩pvc 用于 共享文件 rocket.chat 的aws s3文件上传功能 我没有实验成功

这里上传的文件 保存的是本地文件系统 所以用ceph做了一个共享存储

  • 一个是home目录 (这个目测没啥用 其实只有一个上传目录应该就是可以的)
  • 一个是上传目录 
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: chatserver-uploads-pvc
  namespace: rocket
spec:
  storageClassName: rook-cephfs
  accessModes:
    - ReadWriteMany
  resources:
    requests:
      storage: 5Gi
---
apiVersion: v1
kind: PersistentVolumeClaim
metadata:
  name: chatserver-home-pvc
  namespace: rocket
spec:
  storageClassName: rook-cephfs
  accessModes:
    - ReadWriteMany
  resources:
    requests:
      storage: 5Gi

rocket.chat 的deployment文件

需要注意的地方就是Mongodb url 需要2个环境变量

  • MONGO_URL
  • MONGO_OPLOG_URL
apiVersion: apps/v1
kind: Deployment
metadata:
  name: chatserver-deployment
  namespace: rocket
spec:
  selector:
    matchLabels:
      app: chatserver
  replicas: 3
  template:
    metadata:
      labels:
        app: chatserver
    spec:
      containers:
        - name: chatserver
          image: rocket.chat:latest

          securityContext:
            runAsUser: 0
          ports:
            - containerPort: 3000
          env:
            - name: PORT
              value: "3000"
            - name: NODE_ENV
              value: production
            - name: MONGO_URL
              value: mongodb://mongo-replicate-0.mongodb-service:27017,mongo-replicate-1.mongodb-service:27017,mongo-replicate-2.mongodb-service:27017/rocketchat?replicaSet=rs0&readPreference=nearest&w=majority
            - name: MONGO_OPLOG_URL
              value: mongodb://mongo-replicate-0.mongodb-service:27017,mongo-replicate-1.mongodb-service:27017,mongo-replicate-2.mongodb-service:27017/local?eplicaSet=rs0
            - name: HOME
              value: /home
            - name: ROOT_URL
              value: https://chat.xxxx.com
            - name: Accounts_AvatarStorePath
              value: /app/uploads
          volumeMounts:
            - name: chatserver-uploads
              mountPath: /app/uploads
            - name: chatserver-home
              mountPath: /home
      dnsConfig:
        nameservers:
          - 1.1.1.1
          - 8.8.8.8

      volumes:
        - name: chatserver-uploads
          persistentVolumeClaim:
            claimName: chatserver-uploads-pvc
        - name: chatserver-home
          persistentVolumeClaim:
            claimName: chatserver-home-pvc

然后就是对外访问的service 和 ingress

  • service yaml 文件

    apiVersion: v1
kind: Service
metadata:
name: chatserver-service
namespace: rocket
spec:
clusterIP: 10.43.53.226
ports:
- port: 3000
  protocol: TCP
  targetPort: 3000
selector:
  app: chatserver
sessionAffinity: None
type: ClusterIP

  • ingress yaml 文件

    apiVersion: extensions/v1beta1
kind: Ingress
metadata:
annotations:
  nginx.ingress.kubernetes.io/proxy-body-size: 200m
name: chat
namespace: rocket
spec:
rules:
- host: chat.xxxx.com
  http:
    paths:
    - backend:
        serviceName: chatserver-service
        servicePort: 3000
tls:
- hosts:
  - chat.xxxx.com
  secretName: cert-chat

然后登陆后台 修改一下上传文件的存储位置 就可以了
