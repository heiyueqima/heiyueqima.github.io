  3. 面向对象的设计原则

    • 模块化

      现在软件系统通常包含一些不同的组件 为了使整个系统正常工作，这些组件必须正确的合作。恰当的组织这些组件,才能保证他们合作正常。模块化指的是一种组织原则。 不同的组件归为不同的功能单元。

      模块化易于测试和调试的。一个完整系统的错误，可以追踪到相对独立的特定组件。这样会提高软件系统的健壮性。

    • 抽象化

      抽象化是指从一个复杂的系统中提炼出最基础的部分。描述系统的各个而部分设计给这些部分命名和解析他们的功能。将抽象模式应用于数据结构的设计就产生了抽象数据类型ADT. ADT是数据结构的数学模型。他规定了数据存储的类型，支持的操作，和操作参数的类型。

- 封装 

    软件不应该显示其各自的实现的内部原理，封装的主要优点之一就是他给程序员实现组件的自由，而不同关系程序员写的其他依赖于这些内部代码的程序。程序设计者对于组件的唯一约束是为这些组件保持公共接口，其他程序设计者将会编写依赖于该接口的代码。封装提供了健壮性和适应性。

  3. 软件开发之设计阶段

    对于面向对象编程设计也许是软件开发过程中最重要的阶段。因为子啊决定如何吧程序的工作分成若干个的设计步骤中。我们决定了这些类的交互方式，将要存储逇数据和将要执行的功能。

    对计划进行简单的分类

    • 责任

      把这些工作分为不同的角色。他们有各自不同的责任。试着用行为动词描述责任。这些角色将成为程序的类

    • 独立

      尽可能的独立于其他的类的前提下规定每个类的工作。细分各个类的责任，这样每个类在程序的某个方面就有自主权，把数据作为那些需要控制和访问这些数据的类的实例变量

    • 行为

      仔细切精确的为每个类定义行为，这样与他进行交互的其他可以很好的理解这个由类执行的动作结果，这些行为将定义该类的执行的方法，并且，类的接口是一系列的行为，因为这些类构成了其他代码与类中对象交互的方法。

      设计类的几重常见方法

    • CRC卡

    • UML图

      思考? 自己在编写程序的过程中为什么程序杂乱不堪，没有条理，大概就是没有经过这个设计类的过程。 UML图势在必学。

  4. 软件开发之伪代码

    在设计过程的中间步骤，同城需要程序设计者通过一种专门为人准备的方法来描述算法。这种描述就是伪代码。伪代码不是计算机程序，但是比平常的文章更加结构化。伪代码是自然语言和高级编程结构的混合，用于描述隐藏在数据结构和算法实现之后的主要编程思想。

  5. 软件开发之编码风格和文档

    书里使用的是PEP8

    • python代码块通常锁紧4个空格。

    • 标识符要有意义

      • 类： 首字母大写的单数名词。有多个单次的时候 应该遵循驼峰法命名

      • 函数： 包括成员函数 应该小写，多个单次使用下划线连接

      • 标识某个对象： 包括参数，实例变量，本地变量 他们的名字应该是一个小写的名词

      • 常量： 全部使用大写字母并用下划线连接

      • 封装变量： 下划线开头的变量 是内部使用的变量

    • 注释

      对于比较难以理解的代码块加上注释

    • 文档

      遵循docstring的方法进行文档的编写

  1. 软件开发之调试与测试

    测试是通过实验 检验程序的正确性的过程，调试是跟踪程序的执行并在其中发现错误的过程。

    • 测试

      详细的测试计划是编写程序最重要部分，用所有可能输入检验程序的正确性通常是不行的。所以我们应该用具有代表性的数据子集来运行程序。保证每个方法都至少被执行一次。更好的是每条代码都被执行一遍。

      考虑程序数据的特殊情况

      例如测试一个整数序列的排序方法

      • 序列具有0长度
      • 序列只有一个元素
      • 序列中的所有元素都是相同的
      • 序列已经排序
      • 序列已经反向排序
    测试策略

    - 自上而下的测试 

        从层次结构的顶部向底部进行测试。他通常用于连接存根，一种用桩函数代替了底层的组件启动技术。桩函数是一种模拟原函数组件的替换技术。例如函数A调用的函数B获取文件的第一行 我们可以用返回固定的字符串的桩函数代替B

    - 自下而上的测试

        从低级的组件向更高级的组件进行。
        首先测试不调用其他函数的底层函数，其次测试只调用底层函数的函数。一个不依赖其他类的类可以在依赖前者的其他类之前被测试，通常这种测试的形式，称为单元测试。

        python的 **unittest** 提供了更强大的测试功能，
- 调试

    最简单的方法包括打印语句来跟踪程序执行过程中的变量值。这种方法有一个问题是，最后需要删除或者注释掉打印语句

    一种更好的办法是使用调试器运行程序，调试器是一个专门用于控制和监视程序执行的环境。调试器的基本功能是断点，程序在每个断点处停止，程序终止的时候可以查看程序的变量的当前值。
